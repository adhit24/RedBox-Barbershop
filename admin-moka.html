Setup Guide

Ikuti langkah-langkah ini secara berurutan untuk mengaktifkan integrasi Moka.

1 Jalankan Schema Migration

Buat tabel-tabel baru di Supabase yang dibutuhkan sistem Moka (schedules, outlets, services, dll).

Opsi A — satu perintah terminal:

cd /path/to/project && node server/run-migration.js

Opsi B — paste isi server/moka_integration_schema.sql ke Supabase → SQL Editor → Run

2 Daftar Aplikasi di Moka Developer Portal

Buka developer.mokapos.com → My Apps → Create New App.

Isi Redirect URI dengan:

https://your-domain.vercel.app/api/moka/callback

Salin CLIENT_ID dan CLIENT_SECRET ke server/.env

3 Set MOKA_OUTLET_ID

Di Moka Dashboard → Settings → Outlet Info, salin Outlet ID (angka) ke server/.env sebagai MOKA_OUTLET_ID.

Juga set moka_outlet_id di tabel outlets Supabase untuk setiap cabang.

4 Authorize OAuth (sekali saja)

Setelah .env diisi dan server direstart, klik tombol di bawah atau akses URL ini di browser:

Setelah login di halaman Moka, token disimpan otomatis dan sync dimulai.

5 Map Item Moka ↔ Layanan Website

Buka tab Item Mapping, klik "Load dari Moka", lalu sambungkan setiap layanan website ke produk Moka yang sesuai. Klik "Simpan Mapping".

Mapping ini dipakai saat order dari website dikirim ke Moka agar produk yang muncul di kasir sudah benar.

6 Test End-to-End

Buka website → lakukan booking test → cek apakah order muncul di Moka POS kasir.

Dari sisi GoShow: lakukan transaksi di Moka → kembali ke tab Sync → klik "Pull dari Moka" → cek apakah slot kapster terblokir di website.

💡 Checklist: ENV vars set? Schema applied? moka_outlet_id di tabel outlets diisi? Token OAuth ada? Item mapping selesai?
